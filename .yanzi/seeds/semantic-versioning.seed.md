# Name

semantic-versioning.seed

# Purpose

Provide a concise semantic versioning baseline for release decisions.

# Guidance

- Increment MAJOR for breaking changes.
- Increment MINOR for backward-compatible features.
- Increment PATCH for backward-compatible fixes.
- Keep version intent explicit in release notes.

# Constraints

- Seed is informational and non-executable.
- No references to other artifacts.
